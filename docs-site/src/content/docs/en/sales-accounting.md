## Customer quotes and offers

Quotes and offers include products, quantities, prices, discounts, and terms. Use the catalog to start from consistent data and always check totals, margins, and validity before sending a document.

The free-text **Description** field provides a short identifier for customer quotes, offers, and orders. You can enter it in the document details section; lists show it immediately after the code so you can recognize the content without opening the record. When you select a quote's winning candidate, the generated offer inherits the description, and the order created from that offer inherits it again. While the document status allows edits, you can change the derived document's description without changing its source.

In dialogs that show **Version History** or **Revisions**, the section appears at the top of the dialog content (no longer as a side panel). You can search the list with the magnifying-glass button, always see three row slots (left empty when fewer than three revisions exist), and select an entry with radio buttons. For quotes and offers, save history (**Versions**) remains available from the link at the bottom of the revisions section.

Customer quotes, customer offers, and supplier quotes also have an immutable **Revisions** history. Praetor creates a revision on an actual **Draft** to **Sent** transition only when business data differs from the last sent snapshot; resending unchanged content does not consume a new number. The revision code is stored separately from the document code but is always displayed beside it, for example `OFF_26_001 REV2`, and is searchable in the same **Code** column. The counter is independent for each document and never reuses numbers: after `REV4`, restoring `REV3` and sending different content produces `REV5`.

In the dialog's top section, **Revision history** lists sent snapshots in a scrollable timeline. Selecting a revision shows its date, author, and read-only preview and clears any version preview, and vice versa. **Restore revision** is available only while the current document is still in **Draft**: it first saves the current state in version history, then copies the snapshot without lowering the reserved counter. Restore is also blocked when downstream orders or documents exist, or when it would remove or repoint a supplier line still referenced by a customer document. **Version history** (saves) opens from the link under the revisions list.

In customer and supplier quote dialogs, customer offer dialogs, and customer and supplier order dialogs, line items use Praetor's standard table. The **Items** section keeps the same heading as the dialog's other sections and, when the document allows it, shows **Add item** on that same row. You can sort and filter columns, resize and drag them into a new order, export data, adjust the text size, and change the page size without changing the document order. Decreasing or increasing the text size also adapts row height, cell spacing, and inline controls to the selected size. Quantities, durations, percentages, and amounts are right-aligned like totals, while products, links, and notes remain left-aligned. Values in the **Margin** column are highlighted in green like the summary. New rows do not prefill numeric values: quantities, durations, costs, prices, MOL percentages, and discounts remain empty and show a numeric placeholder in the expected format (`0` or `0,00`) until you enter them or select a source that supplies them. If duration is blank, it stays blank and is normalized to **Months** on save; select **Years** after entering a numeric value. On save, Praetor validates every document row, including rows hidden on other table pages. When a new row exceeds the current page limit, the table automatically opens the page containing the new item. Newly added rows and rows with invalid required numbers remain visible even when filters are active, so you can always complete them. The **Columns** menu remains available for order and view management, but these tables do not allow columns to be hidden. Fields remain editable directly in their cells; each line's notes are in the **Notes** column, which is wider than the other editorial columns, and its editor remains single-line without changing the line-item row height. Actions, including removal, are available from the **…** menu. On narrow screens the table scrolls horizontally.

In create dialogs, the **Code** field for commercial and accounting documents can be left blank: Praetor assigns the code automatically from the template configured under **Administration → General settings → Document codes**. Users who can create that document type see a preview of the next generated code in the empty field, even if they cannot manage the templates. Enter a value only when you need a manual override; that code is preserved on later updates. For customer quotes, customer offers, and supplier quotes, a newly entered or renamed code can contain up to 100 characters using letters, numbers, underscores (`_`), and hyphens (`-`); historical codes with other characters remain accessible and can be saved unchanged or renamed. Automatic sequences are global for all users and separated by module and year. When a document is derived from another document, such as customer quote -> offer -> order -> invoice or supplier quote -> order -> invoice, the automatic code inherits the source year and sequence. Customer invoices inherit the chain code only for the first invoice linked to an order; later invoices for the same order use the module's sequential counter. Sequence gaps are expected when a chain reserves a counter that has no previous documents in the target module.

Each quote line includes a **Duration** column, placed between **Quantity** and **Cost**: it sets how long the service runs. A selector next to the value lets you pick **Months** or **Years** using the same *value / unit* format as quantity. Duration always uses the displayed number without converting between units: **1 year** multiplies by 1, while a multiplier of 12 requires **12 months**. **Total cost** and **Revenue** are computed as *unit value × Quantity × displayed Duration*. For one-off items leave **Duration = 1 month** or select **N/A**.

The line order is stored with the quote and does not change during later edits. While the quote is editable, drag a line by its handle to place it where you want; with the handle focused, you can also use **Up/Down** to move it one position or **Home/End** to move it to either end. The new order becomes permanent when you save the quote and is also retained in version history.

**Duration** flows through the whole document chain: when you convert a quote into an offer and then into a sales order, the lines keep both the value and the unit (months or years) you set. Every document created after this update applies the displayed-value rule, so converting a historical document can produce a different total in the new document. **Customer invoices** also carry the Duration column and multiply the taxable amount, VAT, and total by the displayed value; on invoices the duration is set manually, since invoice lines are not copied automatically from an order.

Duration applies to **every line**, regardless of the quantity unit (**Hours**, **Days**, or **Unit**): the **Months** / **Years** / **N/A** selector multiplies totals by the displayed number. Quantity units are descriptive too: switching between **Hours** and **Days** preserves the numeric cost and price without applying ×8 or ÷8. Selecting **N/A** disables the numeric field and keeps the multiplier at 1.

Documents saved before this update automatically retain their historical totals: a duration in years continues to use the former month multiplier, and a day-labelled product cost retains the former ×8. Praetor does not rewrite the stored duration, unit, cost, or price; compatibility is attached internally to the lines and remains in effect when a historical document is reopened, edited, or restored from version history. New lines added to a historical document follow that document's contract, while a new document — including a copy or conversion — uses the current rule.

Every **quote**, **offer**, and **customer-order** line exposes its unit **Cost** and **Sale price**. Changing either value makes Praetor calculate **MOL** automatically with `MOL = (Sale price − Cost) / Sale price × 100`, immediately updating revenue and margin. MOL is a percentage of revenue: when the sale price is below cost, a negative MOL is shown so a below-cost sale remains visible. You can still edit MOL directly; doing so recalculates the sale price, and the value must remain below 100%, which could not produce a finite price. In **quotes** and **offers**, you can also edit the line's net **Revenue**: Praetor derives the unit sale price from quantity, duration, and the line discount, then updates MOL, margin, and totals without changing cost. Revenue remains disabled until quantity is positive and whenever the line discount is 100%, because those cases cannot be inverted into a unit price. When an item comes from a supplier quote, the effective cost is the net cost after **Discount to Us**, so the supplier discount is not shown as margin on the customer document. With MOL set to 0, before any customer line or global discounts, line revenue equals line cost and the displayed margin is 0. The entered unit sale price stays unchanged when cost is updated, including during synchronization with a supplier quote; MOL is the value that gets recalculated.

Every line also includes a **Discount** column. The line discount is a percentage from 0% to 100% and reduces the line revenue after quantity and duration are applied: `Net revenue = Unit sale price × Quantity × Duration × (1 − Discount / 100)`. The line **Margin** uses this net revenue, while cost is unchanged. Any global discount, which still supports either a percentage or a fixed amount, is applied afterwards to the sum of the already-discounted lines. Global percentage discounts must remain between 0% and 100%, including in API requests; fixed-amount discounts remain non-negative and are capped at the subtotal during calculation, so the document total cannot become negative. Sale price and MOL are copied automatically from quote to offer and from offer to order, and remain editable until the document becomes read-only.

In quote, offer, and order summaries, **Subtotal** is the gross amount before discounts. The **Total discounts** row combines every line discount with any global discount and shows the effective average percentage against the gross subtotal in parentheses, while **Total** remains the final net amount. The same rule applies to supplier orders, including both line and global discounts.

The quote list shows code, insertion date, client, gross subtotal, global discount percentage, total discounts, discounted total, margin, MOL, payment terms, due date, and status so the main values can be checked without opening each record.

From each customer or supplier quote's **…** menu, **Duplicate** opens the regular creation dialog prefilled with the original document's commercial data. Nothing is saved until you confirm: the copy starts as a **Draft**, leaves its code blank for the next automatic sequence value, and sets expiration to one month from the current date. Customer quotes copy every variant, reactivate them, and detach supplier sources and downstream documents; if a line depended exclusively on a supplier source, select a product or a new source before saving. Duplicated supplier quotes do not retain the original order, synchronization, or attachments.

The customer quote family follows **Draft → Sent → candidate selection → Offer**. **Expired** is derived from the active variants: the family is expired only when every active candidate has passed its expiration date. An individual expired variant remains visible but cannot be promoted. A non-expired **Sent** quote is content read-only, matching customer offers: to change lines, discounts, notes, or other commercial fields you must move it back to **Draft**; allowed status changes (including back to draft), candidate promotion, and — when the quote is expired — extending the **Expiration Date** remain available. In the view dialog, **Restore quote** appears next to the read-only badge: it is enabled only when the quote can return to draft, otherwise it stays disabled with a tooltip that explains why.

An offer can be created only with **Choose the candidate to promote**; direct status changes and direct offer creation are rejected. The winning variant supplies the lines and commercial conditions, keeps the parent quote code and counter, and is marked **Selected** while every sibling is archived as **Discarded**.

If the linked offer is still **Draft** and has no orders, **Back to Draft** performs the promotion rollback: it deletes that offer, returns the family to Draft, and reactivates every candidate. Once the offer has progressed or has downstream documents, the quote family remains read-only.

Each customer quote can contain one or more **candidate variants** sharing the document code, customer, and status. The form starts with **Variant A** and presents variants like browser tabs: the **+** button immediately after the tabs opens a choice between a new empty variant and a duplicate of the current one, while the name can be edited by double-clicking it, using the pencil, or choosing **Rename** from the right-click menu. The same menu can duplicate the current variant; the trash button on the tab requests deletion with confirmation. Composition-changing actions remain available while the quote is Draft. Terms, channel, expiration, discounts, notes, and lines belong to each variant; the Variants column shows **N/A** when only the base candidate exists and the alternative count when there are at least two, while financial values remain in their dedicated columns.

From **Sent**, use **Choose the candidate to promote**: when there is only one active, promotable candidate, it is promoted directly without opening the comparison; with multiple variants, the comparison shows total, margin, MOL, expiration, line count, and notes. The selected variant creates the offer and remains marked as selected; the others become discarded and read-only. Expired variants, or variants sourcing expired supplier quotes, cannot be promoted. While the offer is still Draft and has no orders, rollback deletes the offer and reactivates every variant.

A client quote is associated with supplier quotes purely through its **product lines** — each line can be sourced from a supplier quote item (there is no separate header field). While any supplier quote a line sources is **Expired**, the client quote shows an indicator and cannot progress to **Sent**, **Offer**, or **Accepted** until you extend that supplier quote's validity.

When creating or editing a quote, supplier items already used by another customer quote, including its saved variants, do not appear in the picker. Other items from the same supplier quote remain available. An item used in one variant remains available to the other variants of the same quote. In the open variant, items already selected by another line instead appear greyed out and cannot be selected again; unlinking or removing the line makes the item immediately available.

Each picker option starts with the supplier quote code in square brackets, for example `[SQ-001]`.

When you select a supplier quote item, the client line also inherits its **Duration**, preserving the displayed value and unit (**Months**, **Years**, or **N/A**); you can then customize it on the client document.

When creating or editing a **quote** or an **offer**, each **Products / Services** row's **…** menu contains shortcuts for opening the linked **Supplier Quote** and **product record**. On **client orders** and **client invoices**, the shortcut opens the row's linked **product**. Use them to inspect the record on its pre-filtered page in a new browser tab, without closing or changing the document in progress. The actions are available when you have permission to access the destination view; if the row has no record to open, they remain visible but disabled. On the destination page the filter is applied through the column's native filter (**Code**), so it stays visible and you can clear it from the filter menu to return to the full list. Removing a row asks for confirmation first: clicking the trash action opens a prompt, so an accidental click never drops a product line until you confirm.

When horizontal space is limited, the headers and line-item rows of **quotes, offers, orders, and invoices** stay aligned inside a horizontally scrollable area. Drag it on touch devices or use the scrollbar instead of letting document fields collapse; this applies to both client and supplier documents.

In the **Customer Offers** list, the visible date is the **Delivery Date**: it is set when an offer moves to sent status and no longer represents the technical record creation date. The table also shows gross subtotal, global discount percent, total discounts, discounted total, margin, MOL, and payment terms so offers can be compared without opening each record.

When a customer offer is accepted, Praetor automatically creates the linked customer order in **Draft**. Lines linked to derived and accepted supplier quotes also generate the related supplier orders, matching the manual conversion path.

Supplier-order automation never trusts a supplier quote code supplied on its own. For API-created customer orders, each supplier-backed line must carry a valid supplier quote item reference. Without **Create supplier orders** permission, that reference must already belong to the accepted source offer for the same customer, and each item may be used no more times than it appears in that offer; otherwise the request is rejected. When updating an order, those users can retain validated supplier references already stored on its lines but cannot add new ones; the stored historical supplier cost remains unchanged and MOL is recalculated from that cost. Restoring a historical version also revalidates every supplier item and refreshes its quote code, product, supplier, net cost, and MOL from current data; if the item no longer exists, restore is blocked. A quote code without an item reference is discarded and cannot trigger procurement.

A non-expired customer offer in **Sent** status can be moved back to **Draft** from the row actions menu or from the **Revert to Draft** button next to the read-only badge in the view dialog, by users allowed to update offers; the change preserves the Delivery Date and all other offer data. For offers in **Accepted** or **Denied** status, the same operation remains limited to Top Manager and admin users: Praetor requires confirmation, lets the user enter a reason, and records the change in history/audit. When restore is not allowed (missing role or a linked sale order), the header button stays visible but disabled and a tooltip explains why. Reverting a terminal status is unavailable once a sale order has already been created from the offer.

Customer offers follow the same expiration model as quotes: every offer has a mandatory **Expiration Date**, and once it passes a **Draft** or **Sent** offer shows the derived **Expired** status (an **Accepted** or **Denied** offer is never shown as Expired). An expired offer is read-only, cannot change status, and cannot be deleted — the only way out is extending the **Expiration Date**, which becomes editable again in the offer form once the offer expires. Expired documents can be isolated with the **Status** column filter, where **Expired** appears as its own option (on offers and on client quotes alike). A candidate whose date has already passed cannot be promoted: extend and save that variant's expiration first, then choose the candidate again.

## Supplier quotes

Supplier quotes help compare purchase costs and conditions. Link rows to the correct products when possible so data remains traceable in later workflows.

The free-text **Description** field provides a short identifier for the supplier quote. You can enter it in **Supplier Information** while creating or editing the document; the list shows it immediately after the quote code.

In the **Supplier Information** section of the **New Supplier Quote** dialog you must link a **Customer**: every supplier quote has to be associated with a customer. The field is mandatory — it carries the `*` indicator like **Supplier** and **Quote Code**, and saving is blocked until a customer is selected; the empty *No customer* option no longer exists. The linked customer is visible both in the quote detail and in the **Customer** column of the list.

The supplier quote **Status** is fully derived and can never be changed manually. A supplier quote that no client document sources is always a **Draft** (and that is when its items appear among the per-line options in the client quote form); once a client quote's lines source it, the status follows the most-advanced sourcing document: **Sent** when that client quote is sent, **Offer** once a client offer is created from it, **Accepted**/**Denied** when the quote or its offer reaches a terminal state. **Expired** appears when the supplier quote's **own Expiration Date** passes, or when the sourcing quote/offer expires. The **Expiration Date** stays editable in any status to renew validity.

The **Items** table makes the purchase pricing chain explicit with the **Product**, **List Price**, **Discount to Us**, **Unit Cost**, **Qty**, **Duration**, and **Total** columns. The `%` symbol is displayed beside each **Discount to Us** cell value, like the currency symbol in **Unit Cost**. On a new line, **List Price** is empty, shows only the numeric `0,00` placeholder, and is required: you must enter it explicitly before saving. Also enter the discount percentage granted by the supplier (capped at 0–100%, since a larger discount would push the cost below zero): Praetor derives the **Unit Cost** as `List Price × (1 − Discount to Us / 100)`, while the row **Total** is `Unit Cost × Qty × displayed Duration`. The Unit Cost field is read-only because it is derived and displayed at currency precision; for Total, Praetor retains any additional decimal places in that derived cost until quantity and duration have been applied, then rounds the final amount to currency precision. A synchronization from a customer document may preserve an explicit net cost that was already rounded; in that case the synchronized cost remains authoritative across later saves that do not change list price or discount. In the **Summary**, the **Subtotal** sums the list prices (`List Price × Qty × Duration`), the **Discount** row highlights the total discount granted by the suppliers, and the **Total** reports the net cost (`Unit Cost × Qty × Duration`). The Discount row appears only when at least one line has a discount.

The **Duration** column sits after **Qty** and works exactly like the one on [client quotes](#customer-quotes-and-offers): it multiplies the row total by the displayed number, with a **Months** / **Years** / **N/A** selector that does not convert the value. It applies to **every line**, regardless of the quantity unit (**Hours**, **Days**, or **Unit**). Selecting **N/A** disables the numeric field and keeps the multiplier at 1.

While the family is **Draft**, each variant keeps an independent snapshot of supplier-quote lines: changing quantity or cost in one variant neither updates the source nor changes the other variants. In **Sent** the quote content is locked; to edit variants you must return to **Draft**. Only when you choose the winning candidate does Praetor synchronize genuine quantity and cost changes from that variant to the supplier quote; the generated offer then resumes the existing bidirectional synchronization (the list price is recomputed while keeping **Discount to Us**). Re-saving a document that still holds older values, for example after changing only notes, never overwrites direct supplier-quote changes; a freshly linked line instead starts from the source's current values. Synchronizing the winner and subsequent offer changes requires the supplier-quote update permission. Conversely, when a supplier quote is edited directly, the client quote/offer lines that use it show a **Data drifted — sync?** button that pulls the latest values back into the line. Lines sourced from a supplier quote that already generated an order — or whose status is **Accepted**, **Denied**, or **Expired** — stay locked, and a supplier quote whose items are used by client quotes, offers, or orders can no longer be deleted, restored to an earlier version, or have its **Quote Code** changed. Its items stay editable in place — pricing, quantity, notes and other details can change, and the client lines that use them pick the new values up through the **Data drifted — sync?** button — but an item used by a client line cannot be removed or repointed to a different product (the other header fields — payment terms, notes, customer, and the always-editable expiration date — stay editable as well). When a client order is created and a line-sourced supplier quote is not in **Accepted** status (only the supplier quote linked to the client document follows its status), no supplier order is generated for it and the response reports a warning.

You can attach the supplier's files (**xlsx**, **pdf**, or **docx**, up to 10 MB each) in the **Attachments** section. Files can be added straight from the **New Supplier Quote** dialog — they are queued while you fill in the quote and uploaded automatically when you save it — as well as later while the quote is still a draft. Attachments can only be changed on draft quotes with no linked order; once the quote leaves draft or an order is created from it the section becomes read-only and existing files can only be downloaded.

## Orders

Customer and supplier orders consolidate operational information. Before confirming, check the record, rows, discounts, payment terms, and links to previous documents.

A sale order created automatically from an accepted offer starts as a **Draft** and stays fully editable (client, rows, discounts, notes, and payment terms) while it is a draft. After confirmation, the **order code** and **client** stay locked while rows, discounts, notes, and payment terms remain editable. A **Denied** order remains fully read-only.

Rows that automatically generated a **supplier order** (identifiable from the *Supplier order* column) stay locked in draft and after confirmation: they cannot be removed, nor can their product or quantity change, so the linked procurement order never falls out of sync. You can still update their sale price, add other rows, and edit the allowed header fields, such as notes, discounts, and payment terms. The *Supplier order* column carries the same quick-view icon as the product column: it is always shown (so rows stay aligned), and when the row is linked to a supplier order it opens that order — already filtered — in a new tab, so you can review it without leaving the order you are editing. When there is no linked supplier order the icon stays visible but disabled, with a tooltip saying so.

**Supplier orders** keep the same column sequence as the quote: **Product**, **List Price**, **Discount to Us**, **Unit Cost**, **Qty**, **Duration**, **Total**, and **Notes**. Quantity shows its unit (**Hours**, **Days**, or **Units**) beside the value, carried from the supplier quote; it can be changed with the quantity while the order is a draft without repricing the line. The `%` symbol appears in the discount cell rather than the column header. List price, discount, quantity, and duration remain editable while the order is a draft; unit cost is a read-only derived value. **Discount to Us** is always limited to the inclusive 0–100% range, including API requests. Once a supplier invoice is created from a supplier order, the source order becomes read-only and can no longer be renamed, edited, deleted, or restored to an earlier version, preserving the invoice link.

When you create an order from a supplier quote (with the dedicated button or through the automatic conversion), each line keeps its list price, discount, quantity unit, and duration value/unit. If the quote contains an explicitly synchronized net cost that differs from the formula because of rounding, the order line instead preserves that cost as its net price without applying a second discount. The order **Total** is computed as `Unit Cost × Qty × displayed Duration`, so it matches the quote. As in the quote, any fractional cents in the discounted cost are retained through the multipliers and the result is rounded to currency precision only at the end; the same rule continues in supplier invoices generated from the order and in reports. The duration stays editable, with the **Months** / **Years** / **N/A** selector, while the order is a draft; choosing **N/A** keeps the multiplier at 1.

## Invoices

Customer and supplier invoices should match actual orders and deliveries. Check taxable amounts, VAT, totals, and references to the linked document.

After an invoice leaves draft status it becomes read-only: it cannot be moved back to draft or deleted. Delete only draft invoices that were created by mistake, before issuing them.

The amount paid cannot exceed the invoice total. When an invoice is set to **paid**, the amount paid must cover at least the full total; otherwise Praetor rejects the save so aging, balances, and reports stay consistent.

Both **customer and supplier invoices** carry the **Duration** column on their lines, with the same **Months** / **Years** / **N/A** selector, and multiply the line total by the displayed number without converting the unit. When you create a **supplier invoice** from a supplier order the duration is carried over so the invoice total matches the order; otherwise it can be edited per line while the invoice is a draft. Selecting **N/A** keeps the multiplier at 1.

Praetor rounds taxable amounts, VAT, costs, and totals to two currency decimals using commercial half-cent rounding: values such as `1,005` become `1,01`. Every decimal shown or entered in the interface uses a comma as the decimal separator and dots only for thousands (for example `1.234,56`), including when the interface language is English.

### Per-line VAT (IVA)

Each customer-invoice line carries its own VAT rate (percent). New lines default to 22% (the standard Italian rate), but you can edit it to reflect reduced rates (10%, 5%, 4%) or exempt lines (0%). The summary panel shows the taxable subtotal, the total VAT, and the grand total (taxable + VAT). Invoices created before this feature was added load with 0% VAT, so their grand total remains unchanged.

If a document comes from an order or offer, use the automatic link to preserve traceability.
